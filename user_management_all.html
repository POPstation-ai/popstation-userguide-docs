ユーザー管理機能 仕様書（Excelアップロード/ダウンロード）

1. 概要

ユーザー管理の効率化のため、既存ユーザー情報の一括ダウンロードおよびExcelファイルでの一括アップロードを実現する。

アップロード時には、追加・更新・削除が行われる。削除は、アップロードファイルに含まれていないユーザーを対象とする。

2. 仕様詳細

2.1 ダウンロード仕様

対象ユーザー：ログインユーザーが操作可能なユーザーのみ（自分自身・テナント管理者は除外）

ログインユーザーの権限 ダウンロード対象ユーザー
テナント管理者 全ユーザー
管理者 管理者・ユーザー管理者・一般ユーザー（・カスタムデータ管理者）
ユーザー管理者 ユーザー管理者・一般ユーザー（・カスタムデータ管理者）
一般ユーザー/データ管理者 操作画面にアクセスできない

注意事項：ダウンロードしたファイルはそのままアップロード可能

補足：ダウンロード対象ユーザー = 操作可能ユーザー（追加・変更・削除）

2.2 アップロード仕様

処理対象判定

対象 処理内容
自分自身 無視＋エラーレポート
テナント管理者 無視＋エラーレポート
操作権限なしユーザー 無視＋エラーレポート
操作権限ありユーザー 通常通り処理（追加 / 更新 / 削除）

処理フロー

  1. ファイルアップロード
  2. フォーマット／データバリデーション
  3. 操作可能なユーザーのみ処理対象とする
  4. 追加・更新・削除処理を実施
  5. エラーレポート生成

2.3 削除対象ユーザーの定義

条件 削除対象になるか
アップロードファイルに存在しないユーザー なる
テナント管理者 / 自分自身 / 操作権限のないユーザー ならない（削除不可のため対象外）

3. バリデーション仕様

実装 チェック項目 説明
☑️ 必須項目の欠落 メールアドレス、アカウント権限など必須項目が空欄
☑️ メールアドレス形式不正 メールアドレス形式でない値
☑️ ファイル内の重複チェック 同一メールアドレスの重複
☑️ 自分自身 / テナント管理者含む ファイル内に含まれている場合は無視しエラー出力
☑️ 操作権限のないユーザー含む 無視しエラー出力

4. エラーレポート仕様

出力形式

  • アップロード後に画面上で出力
  • 処理対象外ユーザー・バリデーションエラーを出力

出力内容例

行番号 メールアドレス エラー内容
12 sample@example.com 自分自身の情報は変更できません
25 admin@example.com テナント管理者の情報は変更できません
30 manager@example.com 操作権限がありません
45 test@@example.com メールアドレスの形式が不正です
50 user@example.com 必須項目「アカウント権限」が入力されていません
