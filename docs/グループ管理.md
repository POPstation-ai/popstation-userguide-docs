# グループ管理

### ユーザーグループとは

POPstation内でユーザーをグループ化し、グループごとに利用できる生成AIとアクセス可能なカスタムデータグループを設定する機能です。ユーザーは複数のグループに同時に所属でき、ユーザーグループは、デフォルトで「全体」グループに所属します。

ユーザーグループ「全体」では、「全体」グループで設定しているAIモデルと「全体共有」カスタムデータグループが利用可能です。



### カスタムデータグループとは

カスタムデータグループとは、カスタムデータフォルダをグループ化し、データへのアクセスを制御するための機能です。

カスタムデータグループ「全体共通」は、「全体」ユーザーグループに所属するユーザーが利用できます。



#### 利用例

利用例をご紹介します。

下の図は、組織内でAIモデル（LLM）とカスタムデータへのアクセスをどのように管理できるかを示す例です。ユーザーは所属するグループのAIモデル（LLM）とカスタムデータにアクセスできます。

図より、A課課長は「課長以上」ユーザーグループと「課長以上」データグループに所属しているため、AIモデル（LLM）「GPT-4o」「Gemini」「Claude」と、「データA」「データB」のカスタムデータにアクセスできます。

しかし、ユーザー個別に設定された利用制限が優先されるため、所属グループの権限が制限される場合があります。

A課課長の場合、「ユーザー制限」でGeminiの利用制限とカスタムデータグループ「A課」の利用が制限されているため、実際に利用できるのは、モデルは「GPT-4o」「Claude」、カスタムデータグループは「課長以上」「全体共通」となります。

このように、POPstationではグループ設定と個別設定を組み合わせることで、ユーザーの役割や権限に応じた最適なアクセス制御を実現します。
